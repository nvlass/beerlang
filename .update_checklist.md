# Update Checklist for Major Progress

When completing significant work, remember to update these files:

## After Every Prompt ✅
- [ ] **prompt_history.md** - Append the prompt and response summary

## After Completing Major Features ✅
- [ ] **PROGRESS.md** - Update current status, test counts, new components
- [ ] **TODO.md** - Mark completed tasks, update next steps
- [ ] **Git commit** - Commit the changes with descriptive message

## Before Ending Session ✅
- [ ] Review all three files (PROGRESS.md, TODO.md, prompt_history.md)
- [ ] Ensure they're in sync with actual implementation status
- [ ] Update dates if needed

---

**Why this matters:**
- Keeps documentation accurate for future sessions
- Helps collaborators understand progress
- Makes it easy to resume work later
- Creates a clear project history
