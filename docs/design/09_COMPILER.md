# Compiler

The compiler transforms S-expressions (produced by the reader) into bytecode for the VM. It handles literal compilation, special forms, lexical scoping, closure capture, macro expansion, tail call optimization, and exception handling.

## Compiler Structure

```c
struct Compiler {
    BytecodeBuffer* bytecode;      // Output bytecode buffer
    ConstantPool*   constants;     // Constant pool (with deduplication)

    LexicalEnv*     env;           // Current lexical environment
    CaptureList*    captures;      // Current fn's capture list (NULL if not in fn)
    RecurTarget*    recur_target;  // Nearest enclosing loop or fn for recur

    bool            in_tail_pos;   // Currently in tail position?
    int             local_count;   // Total locals in current function

    bool            error;         // Error flag
    char            error_msg[512];// Error message buffer
    const char*     filename;
    int             line;
    int             column;
};
```

### Lexical Environment

```c
struct LexicalEnv {
    LexicalEnv* parent;        // Parent environment
    Value       bindings;      // HashMap: Symbol -> local index (fixnum)
    bool        is_function;   // Function boundary?
    int         base_index;    // Base local index for this scope
};
```

Local lookup walks up the chain of environments but stops at function boundaries
(`is_function == true`). Variables beyond a function boundary must be captured
via the closure mechanism instead.

When `is_function` is true, `base_index` resets to 0 (new stack frame). Otherwise
it inherits from the parent: `parent->base_index + hashmap_size(parent->bindings)`.

### Bytecode Buffer

```c
struct BytecodeBuffer {
    uint8_t* code;       // Bytecode array
    size_t   length;     // Current length
    size_t   capacity;   // Allocated capacity (grows by doubling)
};
```

### Constant Pool

```c
struct ConstantPool {
    Value constants_vec;  // Vector of constant values
    Value const_map;      // HashMap: Value -> index (for deduplication)
};
```

Adding a constant checks `const_map` first. If the value already exists, the
existing index is returned. Otherwise, a new entry is appended to
`constants_vec` and recorded in `const_map`.

### Capture List

```c
struct CaptureEntry {
    Value symbol;        // The captured symbol
    bool  from_closure;  // true = parent accesses via OP_LOAD_CLOSURE (nested closure)
    int   parent_idx;    // local index or closure index in parent
};

#define MAX_CAPTURES 256
struct CaptureList {
    CaptureEntry entries[MAX_CAPTURES];
    int count;
};
```

### Recur Target

```c
struct RecurTarget {
    uint32_t jump_pc;            // PC to jump back to (start of body)
    int      first_binding_index;// local index of first binding/param
    int      n_bindings;         // number of bindings (arity check for recur)
};
```

### Compiled Code (output)

```c
struct CompiledCode {
    uint8_t* bytecode;   // Generated bytecode (owned)
    size_t   code_size;  // Size of bytecode
    Value    constants;  // Vector of constants (owned)
    int      n_locals;   // Number of local variables
    int      arity;      // Function arity (-1 for top-level)
};
```

## Operand Encoding (CRITICAL)

All multi-byte operands are little-endian. The operand sizes are:

| Instruction | Operand | Size | Emitter |
|---|---|---|---|
| `OP_PUSH_INT` | signed integer | 8 bytes | `emit_int64` |
| `OP_PUSH_CONST` | constant pool index | 4 bytes | `emit_uint32` |
| `OP_LOAD_VAR` / `OP_STORE_VAR` | constant pool index (symbol) | 2 bytes | `emit_uint16` |
| `OP_LOAD_LOCAL` / `OP_STORE_LOCAL` | local slot index | 2 bytes | `emit_uint16` |
| `OP_LOAD_CLOSURE` | closure slot index | 2 bytes | `emit_uint16` |
| `OP_CALL` / `OP_TAIL_CALL` | argument count | 2 bytes | `emit_uint16` |
| `OP_ENTER` | additional locals count | 2 bytes | `emit_uint16` |
| `OP_JUMP` / `OP_JUMP_IF_FALSE` | relative offset | 4 bytes (signed) | `emit_int32` |
| `OP_PUSH_HANDLER` | absolute catch PC | 4 bytes | raw `bytecode_emit_uint32` |
| `OP_MAKE_CLOSURE` | multiple fields | 12 bytes total | raw emit (see below) |

`OP_MAKE_CLOSURE` encoding: `code_offset(u32) n_locals(u16) n_closed(u16) arity(i16) name_idx(u16)`.

## Compilation Pipeline

The public entry point is `compile()`:

1. **Reset** compiler state (bytecode buffer, constant pool, environment, error flag).
2. **Create** a top-level lexical environment (`is_function = false`).
3. **Call** `compile_expr(form, false)` — top-level is NOT in tail position.
4. **Emit** `OP_HALT`.
5. **Return** a `CompiledCode*` (transfers ownership of bytecode and constants) or `NULL` on error.

### compile_expr dispatch

`compile_expr` dispatches on the form's type:

- `nil`, `true`, `false` -- `compile_literal`
- fixnum, character -- `compile_literal`
- string, keyword, bigint -- `compile_literal` (goes through constant pool)
- symbol -- `compile_symbol`
- cons (list) -- `compile_list`
- vector -- `compile_vector_literal`
- hashmap -- `compile_hashmap_literal`

## Literal Compilation

```
nil          → OP_PUSH_NIL
true         → OP_PUSH_TRUE
false        → OP_PUSH_FALSE
fixnum       → OP_PUSH_INT <int64>
string/kw/bi → OP_PUSH_CONST <uint32 constant-pool-index>
```

All fixnums use `OP_PUSH_INT` with a full 8-byte operand (no small-integer
optimization currently). Strings, keywords, and bigints are added to the
constant pool and loaded with `OP_PUSH_CONST`.

## Symbol Resolution

`compile_symbol` resolves a symbol through three mechanisms, in order:

1. **Local lookup** — walks up `LexicalEnv` chain (stopping at function
   boundaries). Emits `OP_LOAD_LOCAL`.
2. **Closure capture** — `capture_lookup` walks past the current function
   boundary into parent scopes. If the variable is found, it is added to
   the current function's `CaptureList` and emits `OP_LOAD_CLOSURE`.
3. **Namespace var** — adds the symbol to the constant pool and emits
   `OP_LOAD_VAR` with the constant pool index. Resolution happens at runtime.

### Closure Capture Details

`capture_lookup` finds the function boundary env, then walks into parent scopes:

- If the variable is found in the immediate parent function's scope
  (`fn_boundaries_crossed == 0`): records `from_closure = false` with the
  parent's local index.
- If the variable is past another function boundary
  (`fn_boundaries_crossed > 0`): records `from_closure = true`. When
  `compile_fn` later emits capture-push code, it resolves `from_closure`
  entries by looking up the symbol in the (now-restored) parent context,
  potentially triggering the parent to capture the variable as well.

## List Compilation (Special Forms and Calls)

`compile_list` checks the head of the list:

1. **Special form check** — if the head is a known special form symbol, dispatch
   to the appropriate compiler function.
2. **Macro check** — if the head symbol resolves to a var with `is_macro` set,
   expand the macro at compile time and recursively compile the result.
3. **Function call** — compile arguments left-to-right, compile the function
   expression, emit `OP_CALL` or `OP_TAIL_CALL` (if in tail position).

### Implemented Special Forms

| Form | Compiler Function |
|---|---|
| `quote` | Pushes quoted form as literal constant |
| `quasiquote` | Expands to `list`/`concat` calls, then compiles |
| `if` | `compile_if` |
| `do` | `compile_do` |
| `def` | `compile_def` |
| `defmacro` | `compile_defmacro` |
| `fn` | `compile_fn` |
| `let*` | `compile_let` |
| `loop` | `compile_loop` |
| `recur` | `compile_recur` |
| `throw` | `compile_throw` |
| `try` / `catch` | `compile_try` |

**Not yet implemented:** `yield`, `var` (`#'`), `.` (interop), `disasm`, `asm`.

### Forms implemented as macros (in `lib/core.beer`)

`defn` (including multi-arity), `let` (with destructuring), `when`, `when-not`,
`and`, `or`, `cond`, `->`, `->>`, `if-not`, `if-let`, `when-let`, and many
stdlib functions.

The compiler primitive is `let*` (simple symbol bindings only). The `let` macro
expands vector destructuring into `let*` with `nth`/`rest` calls.

## Special Form Compilation Details

### if

```
compile test
OP_JUMP_IF_FALSE +offset_to_else   ; relative int32, peeks (doesn't pop)
OP_POP                              ; pop truthy condition
compile then_branch                 ; inherits tail position
OP_JUMP +offset_to_end             ; relative int32
; else target:
OP_POP                              ; pop falsy condition
compile else_branch                 ; inherits tail position (or OP_PUSH_NIL if omitted)
; end target:
```

Jump offsets are calculated relative to the PC after reading the offset (i.e.,
`target - instruction_start - 5` where 5 = 1 byte opcode + 4 bytes operand).
Offsets are emitted as placeholders (0) and patched via `bytecode_patch_int32`
after the target is known.

### do

Compiles all expressions sequentially, emitting `OP_POP` after each non-last
expression. The last expression inherits tail position. Empty `do` emits
`OP_PUSH_NIL`.

### def

```
compile value_expr
OP_STORE_VAR <symbol-constant-pool-index>   ; stores value, leaves it on stack
```

`OP_STORE_VAR` uses a uint16 operand pointing to the symbol in the constant pool.
The value remains on the stack as the result of `def`.

### defmacro

1. Builds `(fn name [params] body...)` and compiles it via `compile_fn`.
2. Stores the function via `OP_STORE_VAR`.
3. Calls the native `set-macro!` function with the name symbol to mark the var
   as a macro.
4. Returns the var's value as the result.

### fn

Function compilation uses an **inline body with skip-jump** approach rather than
creating a nested compiler. The function body is emitted inline into the same
bytecode buffer, with a `OP_JUMP` before it to skip over during normal execution.

**Compilation steps:**

1. Parse optional name, parameter vector (including `&` rest params), and body.
2. Emit `OP_JUMP` with placeholder offset (to skip over the body).
3. Record `fn_code_offset` (where the function body starts).
4. Save parent env and captures; create new function-boundary env and capture list.
5. Add parameters as locals (required params, then rest param if variadic).
6. If named, add name as an extra local after params.
7. Emit `OP_ENTER` with placeholder (additional locals count).
8. If named, emit `OP_LOAD_SELF` + `OP_STORE_LOCAL` to populate the self-reference slot.
9. Set up `RecurTarget` pointing to current PC (after ENTER and self-ref setup).
10. Compile body as implicit `do`, with last expression in tail position.
11. Emit `OP_RETURN`.
12. Patch `OP_ENTER` operand with `local_count - total_params` (additional locals only).
13. Patch the skip `OP_JUMP` to point past the body.
14. Snapshot captures and restore parent env/captures.
15. If no captures: create function object, add to constant pool, emit `OP_PUSH_CONST`.
16. If captures exist: emit `OP_LOAD_LOCAL` or `OP_LOAD_CLOSURE` for each captured
    variable (in the parent's context), then emit `OP_MAKE_CLOSURE` with all metadata.

**Variadic functions:** The `&` symbol in the parameter vector triggers variadic
handling. Arity is encoded as `-(required_count + 1)`. The VM packs extra
arguments into a list for the rest parameter.

### Named Function Self-Recursion

Named functions `(fn name [params] body)` support self-recursion through a
self-reference local variable. When a function has a name, the compiler:

1. Adds the name as an extra local slot (after all parameters)
2. Emits `OP_LOAD_SELF` + `OP_STORE_LOCAL` after `ENTER` to populate that slot
   with the current function value

`OP_LOAD_SELF` pushes the current call frame's function onto the stack. This
makes the function available as a local variable inside its own body, enabling
recursive calls without relying on namespace lookup.

**Two kinds of "foo" in `(def foo (fn foo [x] ...))`:**

```clojure
(def foo (fn foo [x] (+ x (foo (- x 1)))))
;;   ^         ^               ^
;;   |         |               |
;;   |         |               +-- Resolves to LOCAL (slot 1) via OP_LOAD_LOCAL
;;   |         +-- Self-reference local (slot 1, after param x at slot 0)
;;   +-- Namespace var (stored via OP_STORE_VAR after fn is created)
```

- **Local `foo`** (slot 1): The self-reference inside the fn body, populated by
  `OP_LOAD_SELF` at function entry. Used for recursion.
- **Namespace `foo`**: The var created by `def`. Points to the same function but
  resolved via `OP_LOAD_VAR` (namespace lookup).

The local takes priority during symbol resolution (locals are checked before
namespace vars). This is important because:
- The local is available immediately when the function starts executing
- The namespace var doesn't exist yet during the initial `def` evaluation
- Local access (`OP_LOAD_LOCAL`) is faster than var lookup (`OP_LOAD_VAR`)

**With different names:**

```clojure
(def foo (fn bar [x] (bar (- x 1))))
;;   ^         ^       ^
;;   |         |       +-- Resolves to LOCAL (self-reference)
;;   |         +-- Self-reference name (only visible inside body)
;;   +-- Namespace var name (visible everywhere else)
```

Here `bar` is the self-reference (local), `foo` is the namespace var. Inside
the body, `(bar ...)` uses the local; `(foo ...)` would use namespace lookup.

**`defn` always creates both with the same name:**

```clojure
(defn foo [x] (foo (- x 1)))
;; expands to: (def foo (fn foo [x] (foo (- x 1))))
;; Inside body: foo resolves to local (self-ref), not namespace var
```

**Note:** This is distinct from `recur`, which reuses the current frame (tail
call optimization) rather than creating a new call frame. Named self-calls
create new frames and work in non-tail position (e.g., tree recursion in
fibonacci), while `recur` only works in tail position.

### let* (compiler primitive)

`let*` supports sequential bindings where each binding can reference previous ones.

If compilation occurs **outside a function frame** (no `is_function` env in the
chain), `let*` wraps the entire form in a synthetic zero-arity function: emits
`OP_JUMP` to skip the body, `OP_ENTER`, bindings and body, `OP_RETURN`, then
patches the jump, creates a function object via `OP_PUSH_CONST`, and immediately
calls it with `OP_CALL 0`. This is the **need_frame pattern**, also used by
`loop` and `try`.

Inside a function frame, bindings simply extend the current lexical environment
with `OP_STORE_LOCAL` for each binding.

### loop / recur

**loop** is similar to `let*`: it creates bindings and sets up a `RecurTarget`
pointing to the start of the body (after bindings are stored). It also uses the
need_frame pattern at top level.

**recur** validates:
- It is inside a `loop` or `fn` (has a `RecurTarget`).
- It is in tail position.
- Argument count matches `n_bindings`.

Then it compiles all arguments, stores them in reverse order (LIFO) to the
binding slots, and emits `OP_JUMP` back to `recur_target->jump_pc`.

### throw

Compiles the expression and emits `OP_THROW`.

### try / catch

```
(try body... (catch e catch-body...))
```

Uses the need_frame pattern at top level. Compilation:

1. Emit `OP_PUSH_HANDLER` with placeholder absolute PC for the catch block.
2. Compile body expressions (not in tail position inside try).
3. Emit `OP_POP_HANDLER` + `OP_JUMP` to skip past catch.
4. Patch `OP_PUSH_HANDLER` target to current offset (catch block start).
5. Emit `OP_LOAD_EXCEPTION` to get the thrown value.
6. Bind exception to catch variable via `OP_STORE_LOCAL`.
7. Compile catch body.
8. Patch the skip jump.

Note: `finally` is not yet implemented.

## Macro Expansion

Macro expansion happens at compile time using the **temp VM pattern**:

1. Collect unevaluated argument forms from the macro call.
2. Build a mini bytecode sequence: `OP_PUSH_CONST` for each arg, `OP_PUSH_CONST`
   for the macro function, `OP_CALL n_args`, `OP_HALT`.
3. Create a temporary VM (stack size 256), load the code and constants.
4. Run the VM to completion.
5. Extract the result from the stack top (retaining it before freeing the VM).
6. Compile the expanded form recursively.

This same pattern is used by the `apply` and `macroexpand-1` native functions.

### Quasiquote Expansion

`quasiquote` is expanded at compile time into calls to `list` and `concat`:

- `(unquote x)` becomes `x` (evaluated).
- `(unquote-splicing x)` inside a list becomes a `concat` segment.
- Atoms become `(quote atom)`.
- Lists become `(concat (list (qq-expand elem1)) (list (qq-expand elem2)) ...)`.

The expanded form is then compiled normally.

## Vector and Map Literal Compilation

Vector literals `[1 2 3]` compile each element, then load the `vector` native
function via `OP_LOAD_VAR` and call it with `OP_CALL n`.

Map literals `{:a 1 :b 2}` extract entries, compile each key-value pair, load
`hash-map` via `OP_LOAD_VAR`, and call it with `OP_CALL (n_entries * 2)`.

## Function Calls

```
compile arg1        ; left-to-right, NOT in tail position
compile arg2
...
compile fn_expr     ; NOT in tail position
OP_CALL n           ; or OP_TAIL_CALL n if in tail position
```

Arguments are compiled before the function expression. The VM expects the
function on top of the stack, with arguments below it.

## Tail Call Optimization

The compiler tracks tail position through the `in_tail_pos` flag, propagating
it through `if` branches, `do` (last expression), `let*` body, `loop` body,
and `try`/`catch` bodies (when not wrapped in a synthetic frame).

When a function call is in tail position, `OP_TAIL_CALL` is emitted instead of
`OP_CALL`. The VM reuses the current call frame for `OP_TAIL_CALL`.

`recur` achieves tail recursion differently: it stores new values to the
binding slots and jumps back to the loop/fn body start, without any call frame
manipulation.

## Error Handling

Errors are reported via `compile_error(c, fmt, ...)`, which uses `vsnprintf`
into the 512-byte `error_msg` buffer and sets the `error` flag. Once an error
is set, subsequent `compile_expr` calls return immediately (early exit). Only
the first error is recorded.

The error can be checked with `compiler_has_error()` and the message retrieved
with `compiler_error_msg()`.

## Compilation Example

```clojure
;; Source
(defn factorial [n]
  (if (< n 2)
    1
    (* n (factorial (- n 1)))))
```

Macro expansion transforms this to:
```clojure
(def factorial (fn factorial [n]
  (if (< n 2)
    1
    (* n (factorial (- n 1))))))
```

Generated bytecode (simplified, showing the fn body):

```
;; Skip jump (to skip over function body during top-level execution)
JUMP +<offset_past_body>

;; Function body starts here (fn_code_offset):
ENTER 0                          ; 0 additional locals (1 param + 1 name = base)
LOAD_SELF                        ; push current function
STORE_LOCAL 1                    ; store self-ref "factorial" in slot 1

;; recur target is here

;; (< n 2):
LOAD_LOCAL 0                     ; n
PUSH_INT 2
LOAD_VAR <const:"<">
CALL 2

JUMP_IF_FALSE +<offset_to_else>
POP                              ; pop truthy condition

;; then branch: 1
PUSH_INT 1
RETURN

;; else target:
POP                              ; pop falsy condition

;; (* n (factorial (- n 1))):
LOAD_LOCAL 0                     ; n (arg to *)
LOAD_LOCAL 0                     ; n (arg to -)
PUSH_INT 1                       ; 1 (arg to -)
LOAD_VAR <const:"-">
CALL 2                           ; (- n 1)
LOAD_LOCAL 1                     ; factorial (self-ref, NOT LOAD_VAR)
CALL 1                           ; (factorial (- n 1))
LOAD_VAR <const:"*">
TAIL_CALL 2                      ; (* n ...) — tail call!

;; After function body (skip jump lands here):
PUSH_CONST <fn-object>           ; push the compiled function
STORE_VAR <const:"factorial">    ; def stores it in namespace
HALT
```

Note: The recursive `(factorial ...)` call resolves to `LOAD_LOCAL 1` (the
self-reference slot), not `LOAD_VAR`. The outer `(* ...)` is the actual tail
call.

## Jump Resolution

The compiler does **not** use a label system. Instead, it uses **offset
patching**:

1. Emit jump instruction with a placeholder operand (0).
2. Record the bytecode offset where the placeholder was written.
3. After the target code is emitted, calculate the actual offset.
4. Patch the placeholder using `bytecode_patch_int32` (for relative jumps)
   or `bytecode_patch_uint32` (for absolute PCs like `OP_PUSH_HANDLER`).

Relative jump offsets are calculated as: `target_pc - (jump_instruction_pc + 5)`,
where 5 accounts for the 1-byte opcode and 4-byte operand.

## Future Optimizations

> **Note:** The following optimizations are not yet implemented. They represent
> potential future improvements.

1. **Constant folding** — evaluate constant expressions at compile time
2. **Small integer optimization** — use a compact encoding for small fixnums
   instead of always emitting 8-byte `OP_PUSH_INT`
3. **Dead code elimination** — remove unreachable code after returns/jumps
4. **Inline small functions** at call sites
5. **Specialize polymorphic operations** (type-specific math)
6. **Register allocation** for hot functions
7. **Loop unrolling** for known iteration counts
8. **Escape analysis** for stack allocation
9. **Profile-guided optimization** based on runtime data
